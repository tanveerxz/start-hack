# Source Code Structure

## Folder Organization

### `/components`
Reusable React components
- `Header.tsx` - Main header component
- `Footer.tsx` - Footer component
- `Card.tsx` - Generic card component
- `Button.tsx` - Button component

### `/hooks`
Custom React hooks
- `useExample.ts` - Example hook (placeholder)

### `/types`
TypeScript type definitions and interfaces
- Type definitions for your application

### `/utils`
Utility and helper functions
- `helpers.ts` - Common helper functions

### `/constants`
Application-wide constants
- API URLs, pagination settings, messages, etc.

## Quick Start

1. Replace placeholder content with your actual implementation
2. Add new components to `/components` as needed
3. Create custom hooks in `/hooks` for shared logic
4. Define types in `/types` for better type safety
5. Extract common functions to `/utils`

## Naming Conventions

- **Components**: PascalCase (e.g., `UserProfile.tsx`)
- **Hooks**: camelCase starting with `use` (e.g., `useAuthContext.ts`)
- **Utilities/Functions**: camelCase (e.g., `formatDate.ts`)
- **Types/Interfaces**: PascalCase (e.g., `User.ts`)

Happy building! 🚀
